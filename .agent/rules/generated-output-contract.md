# Rule: Generated Output Contract

Governs the structure and content invariants for all files generated by Msingi's `Build-*Md` functions.

## Scope

- **Scripts**: `msingi.ps1`, `msingi.sh` — all `Build-*` functions
- **Output**: Every file in the generated scaffold directory
- **Subagents**: Foundation Maintainer, Context Architect

## Metadata

- **Maintainer**: Context Architect
- **Status**: Stable
- **Contract Version**: 3.9.0

## Context

Both PS7 and Bash scripts must produce byte-identical scaffold files for the same inputs. The generated files are consumed by AI agents and downstream tooling — any schema drift breaks the ecosystem.

## File Invariants

### CONTEXT.md
- **Mandatory Fields**: Project name, project type(s), selected agents, inferred skills, intake profile summary.
- **Ordering**: Architecture section before NFRs, NFRs before stack details.
- **Constraint**: Must include all intake answers (audience, auth, sensitive data, deployment, scale).

### SECURITY.md
- **Auth Section**: Conditional: The authentication/authorization section fires only when `NeedsAuth=true` in the intake.
- **Threat Model**: Must be scoped to the project type (web, API, mobile, desktop — each has distinct threat categories).
- **Requirement**: Derive all security content from intake answers — ensure every section is project-specific.

### WORKSTREAMS.md
- **Agent Stubs**: Mandatory: One workstream stub per selected agent, with role-specific scope hints.
- **Lifecycle State Machine**: Must include the `ACTIVE → BLOCKED → READY_FOR_MERGE → MERGED | FAILED` state machine.
- **Swarm Routing**: Coordinator vs executor permissions must be explicit.

### SESSION.md
- **Context Cost Log**: Mandatory: Section for tracking tokens consumed per session.
- **Token Leverage Note**: Mandatory: Section explaining token throughput degradation over session length.
- **Handoff Format**: Must include Status (Active/Partial/Complete), Last Action, Next Steps, Blockers.

### gotchas.md (per skill)
- **Confidence Metadata**: Mandatory on every entry: `confidence: ●●●●●` through `●○○○○`.
- **Required Fields**: `triggers:` (keywords), `last_seen:` (date or "seeded"), `status:` (ACTIVE/RESOLVED/WONTFIX).
- **Format**: Structured as What/Why/Prevention sections.

### bootstrap-record.json
- **Valid JSON**: Mandatory: Must always be valid JSON (parseable by any standard JSON parser).
- **Required Fields**: Project name, type, selected agents, selected skills, all intake answers.
- **Requirement**: Use strict JSON compliance — ensure valid JSON without comments or trailing commas.
- **Schema Stability**: The `bootstrap-record.json` schema must never change without incrementing the record format version.

### SKILL.md (per skill)
- **Sprint Contract**: Mandatory: Section for mapping acceptance criteria to verification steps.
- **Quick Start**: Must include the category-specific interface and the #1 gotcha from the skill.
- **Research Attribution**: Preserve the harness research attribution when present.

## Parity Enforcement

- **Byte-Identical**: Mandatory: Every `Build-*Md` function must produce identical output byte-for-byte between PS7 and Bash.
- **Timestamp Exclusion**: Timestamps are the only acceptable divergence — use `$(Get-Date)` / `$(date)`.
- **Platform Tag**: The "PS7" vs "BASH" platform tag in headers is an acceptable divergence.

## Verification

- **Diff Test**: Run both scripts with identical inputs, diff all generated files.
- **JSON Validation**: Parse `bootstrap-record.json` with a strict JSON parser after every generation.
- **Section Check**: Grep for mandatory section headers in each generated file.
