# Contributing to Msingi

Thank you for contributing. Msingi exists to make AI agent development accessible to every developer — including those building in constrained environments. Please read this before opening a PR.

---

## Philosophy

Every contribution should make Msingi more useful for more people — especially developers who are not working on high-bandwidth connections with unlimited API budgets.

**We will always accept:**
- New project types (beyond the current 6) with accurate skill pools
- New agent registrations in `agents.json` with correct file, scratchpad, and docs fields
- New skills in `skills.json` with accurate trigger patterns and correct type scoping
- Gotcha seeds for underrepresented skill categories, in the confidence-weighted format
- Bug fixes in `msingi.ps1` or `msingi.sh`
- Improvements to generated file quality (CONTEXT.md, SECURITY.md, SKILL.md, etc.)
- Documentation improvements and translations

**We will not accept:**
- Features that require a network connection, API key, or cloud account
- Features that require Node.js, Python, or any runtime beyond Bash 4+ / PowerShell 7+
- Changes that break the offline-first design
- Dependencies on paid or proprietary services

## Adding a new skill

Skills live in `skills.json`. Each entry must have:

```json
{
  "id": "kebab-case-id",
  "name": "Human Readable Name",
  "category": "auth|data|api|ui|ml|infra|messaging|testing|android",
  "types": ["web-app", "api-service", "..."],
  "baseline": false,
  "trigger": "regex|pattern|for|matching|description"
}
```

Rules:
- `id` must be unique, kebab-case, under 40 characters
- `trigger` is a case-insensitive regex matched against the project description + stack
- `types` limits which project types can infer this skill — use all types only for truly universal skills
- `baseline: true` means the skill is always included for matching types regardless of trigger

For each new skill, also add seeded gotchas to `Build-SkillGotchas` in `msingi.ps1` and in `msingi.sh`. Use the confidence-weighted format:

```markdown
### G-001 · Short title
`confidence: ●●●○○ medium`  `triggers: keyword1, keyword2`  `last_seen: seeded`  `status: ACTIVE`
**What:** What went wrong.
**Why:** Why it happened.
**Prevention:** How to prevent it.
```

## Adding a new agent

Agents live in `agents.json`:

```json
{
  "id": "unique-kebab-id",
  "name": "Display Name",
  "file": "FILENAME.md",
  "scratchpad": "scratchpad-folder-name",
  "docsUrl": "https://docs.example.com/"
}
```

The agent config file format (AGENT.md) is generated by `Build-AgentConfig` in `msingi.ps1`. If the new agent uses a JSON config format rather than Markdown (like Opencode), handle it as a special case in the emit loop.

## Reporting a bug

Open an issue with:
- Which script: `msingi.ps1` (Windows) or `msingi.sh` (macOS/Linux)
- Your OS and shell version
- The project type you selected
- The exact error message or unexpected behaviour
- Whether `--dry-run` reproduces the issue

## Code style

**PowerShell (`msingi.ps1`):**
- Run the 25-test suite in `tests/` before every PR: `python3 tests/test_suite.py`
- All here-strings must close with `"@` at column 0
- No backtick escape sequences inside here-strings (use `$(if...)` patterns instead)
- CRLF line endings — the CI will check

**Bash (`msingi.sh`):**
- `bash -n msingi.sh` must pass with zero errors
- No `set -e` — handle errors explicitly with `|| true` or explicit checks
- Avoid `local` outside functions — use plain variable assignment in main body
- All `(( i++ ))` arithmetic must be followed by `|| true`
- All `grep` calls inside `$()` must be followed by `|| fallback` to handle no-match
